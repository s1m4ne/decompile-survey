# スクリーニング基準: デコンパイル研究 v2（改訂案）

## 目的とスコープ
本スクリーニングは、**機械学習（ML/DL）を用いて**「バイナリ/アセンブリ等から**高級言語のソースコード相当**を復元する、または**デコンパイル擬似コードを“ソースに近づける”**研究」を対象とする。

---

## 採択条件（すべて満たす）

### 条件1: ML/DL を中核に用いている
- Transformer / LLM / language model / neural network / deep learning / machine learning 等を使用
- **重要**: ルールベースや統計のみで、ML/DLが主要要素でない場合は除外

### 条件2: 「デコンパイル（復元）」が主目的である
- 研究の主目的が、以下のいずれかであること：
  - **高級言語ソースの復元**（例: C/C++/Rust 等のコード生成）
  - **デコンパイル擬似コードの修復・高級化**（例: 可読性向上、構造回復、制御構造の回復、型推定、変数名復元、式の簡約など）
- **重要**: デコンパイルが “特徴量抽出や前処理として使われるだけ” の場合は除外（後述）

### 条件3: 入出力の範囲を満たす（どれか）
**入力（いずれか）**
- binary（機械語バイナリ）
- assembly（逆アセンブル結果を含む）
- decompiler output（既存デコンパイラの擬似コード）

**出力（いずれか）**
- 高級言語のソースコード（C/C++/Rust等）
- 擬似コードの修復・高級化（「ソースに近づける」変換を含む）

---

## 除外条件（いずれか該当で除外）

### A. 目的が別タスク（デコンパイルは手段）
以下が主目的で、デコンパイル/復元が副次的（特徴量・前処理・説明用）なら除外：
- matching / retrieval（コード検索・類似度マッチング・clone/similarity）
- identification（関数識別・バイナリ識別・family分類・属性推定）
- SCA（Software Composition Analysis）
- summarization（要約・コメント生成・説明文生成が主）
- vulnerability detection（脆弱性検出・予測）
- patch detection（パッチ検出・差分識別）
- verification（等価性検証・正当性証明・バグ検証）

> 判定ガイド:
> - 研究の評価指標が F1/AUC/検出率/分類精度中心 → 多くは「A」に該当しやすい  
> - 生成物が“コード”でも、目的が検出・識別なら除外

### B. 変換先が「高級言語/擬似コード高級化」ではない
- IR-only（LLVM IR 等の中間表現止まりで、高級言語への変換なし）
- 逆アセンブル結果の整形のみ（disassemblyの可読化・コメント付け等）で、復元がない
- 出力が「特徴量」「埋め込み」「グラフ」「ラベル」などで、コード生成・擬似コード高級化がない
- デコンパイル対象が本スコープ外（例: 量子回路など、ソフトウェアバイナリ復元と別物）

### C. 論文タイプが評価・整理中心で、新規復元手法が主でない
- ベンチマーク、データセット、チュートリアル、サーベイ、ツール紹介が中心で、
  **新規のMLベース復元手法の提案・検証が主ではない**場合は除外  
  （※ただし「新規タスク設定＋強いベースライン提供」等で復元研究として明確に寄与する場合は採択可）

---

## 判定の注意点（運用ルール）

### 1) “decompile” という語の有無で判断しない
- タイトル/アブストラクトに "decompile" があっても、
  **除外条件A〜C**に当たれば exclude

### 2) “復元の主目的” を優先して判定する
- 同じ生成でも、目的が違えば扱いが変わる：
  - 生成コードで **脆弱性検出** → exclude（A）
  - 生成コードで **擬似コード修復/ソース復元** → include（条件2）

### 3) 境界例の取り扱い（迷ったときの決め方）
- **採択寄り（include）**:
  - 変数名復元、型推定、制御構造回復などが主で、擬似コード/ソースを改善している
  - 既存デコンパイラ出力の「ソースに近い変換」をMLで行う
- **除外寄り（exclude）**:
  - バイナリの埋め込み学習が主で、最終成果が検索/識別/検出
  - 解析・検出パイプラインの一部としてデコンパイルを使うだけ

---

## 付録: 判定ラベル（記録用のテンプレ）
- Decision: include / exclude / unsure
- Reason (include): 条件1, 条件2, 条件3 の根拠を1行ずつ
- Reason (exclude): A/B/C のどれに該当するか + 根拠1行
- Note: 境界例なら「何が足りない/どこが曖昧か」を1行
